# Kakeiboku — Agent Instructions

Kakeiboku is a personal finance management platform consisting of:
- **`apps/apiportal/`** — Go (Gin) REST API with JWT auth and PostgreSQL
- **`apps/webportal/`** — Nuxt 3 frontend (work in progress)
- **`docker/`** — Docker Compose for local development

---

## Build & Test

All commands run from `apps/apiportal/` unless noted.

| Task | Command |
|------|---------|
| Build | `make build` |
| Run | `make run` |
| Unit tests (no DB) | `make test` |
| All tests (DB required) | `make test-all` |
| Live reload | `make watch` |
| Docker up (from root) | `make docker-run` |
| Docker down (from root) | `make docker-down` |

Integration tests are named with the `_it_` infix (e.g., `store_transaction_tx_it_test.go`) and require a running PostgreSQL container. `make test` skips them via `-skip "_it_"`.

## Code Generation (run after DB or query changes)

```bash
# Regenerate sqlc Go code from SQL queries
make sqlc

# Regenerate mocks (run after Store or TokenMaker interface changes)
make store-mock
make token-mock
```

- SQL queries live in `pkg/queries/*.sql`; migrations in `pkg/migrations/`
- Generated code goes to `pkg/services/` — **do not edit** files starting with `// Code generated by sqlc`
- Mocks live in `pkg/mock/` — **do not edit** generated mock files by hand

## Architecture

```
cmd/api/main.go          → entry point (loads config, wires server)
internal/server/         → Gin handlers + route registration
internal/middleware/     → Auth (JWT Bearer) and web middleware
internal/validator/      → Custom Gin binding validators (e.g. currency)
pkg/services/            → Store interface + sqlc-generated DB layer + TX helpers
pkg/token/               → TokenMaker interface (JWT impl)
pkg/utils/               → Config, pagination, password hashing, response helpers
pkg/mock/                → GoMock mocks for Store and TokenMaker
pkg/migrations/          → golang-migrate SQL files (numbered sequentially)
```

**Store pattern**: `Store` interface (in `pkg/services/store.go`) embeds `Querier` (auto-generated) and adds transaction methods (`CreateUserTx`, `CreateTransferTx`, `SetSession`). Use `ExecTX` to wrap multi-step DB operations.

**Token pattern**: `TokenMaker` interface supports both JWT and PASETO. Currently JWT is wired in `server.go`. Always use the interface, never the concrete type.

## Conventions

- **Error responses**: Use `utils.ErrorResponse(err)` for all `ctx.JSON` error returns.
- **Common responses**: Use `utils.CommonResponse(msg)` for simple success messages.
- **Auth payload**: Extract from Gin context with `ctx.MustGet(middleware.AuthorizationPayloadKey).(*token.Payload)` in protected handlers.
- **Config**: Loaded via Viper from `app.env`. Duration fields must be valid Go duration strings (e.g. `24h`). See `pkg/utils/configs.go` for the full `Config` struct.
- **Currency validation**: Use the `currency` binding tag (registered custom validator in `internal/validator/currency_validator.go`).
- **Pagination**: Use `utils.PaginationQueryParam` embedded in query-param structs; call `utils.GetPaginationParams(ctx)` in handlers.
- **Migrations**: Create with `make migrate-create name=<name>`. Always provide both `.up.sql` and `.down.sql`.
- **Routes**: All routes are prefixed `/v1`. Protected routes use `middleware.AuthMiddleware(server.Token)`.

---

## Testing Conventions

- Unit tests live in `internal/tests/` and use GoMock + `gofakeit` for fake data.
- `newTestServer(store, tokenMaker)` in `main_test.go` wires a test `Server` without a real DB.
- Use `httptest.NewRecorder()` + `router.ServeHTTP(recorder, req)` for handler tests.
- Integration tests (`_it_`) connect to a real DB via `pkg/services/test/`.
- `gin.SetMode(gin.TestMode)` is set in `TestMain`.

## Environment

Copy `apps/apiportal/app.env.example` to `apps/apiportal/app.env` and fill in values before running locally. The `Makefile` exports all env vars from `app.env` automatically via `include app.env / export`.
